3 Program #1: Matrix-multiplication improved

3.1 Introduction

If you analyse your thread-based parallel implementation of Project 1 (Phase 3), you should notice that the overhead of adding POSIX threads support was not negligible compared to the initial serial code.

You probably needed to initialize a few data structures that were given as arguments to the threads. Then you had to create the threads themselves, before finally waiting for all of them to complete. Threads were necessarily defined as a separate function which processes a chunk of the matrix multiplication, as defined by the received parameters.

Pthreads-based implementations also often suffer from a lack of flexibility. If the number of threads to create is defined as a constant, then the program needs to be recompiled for any new computers in order to match the number of threads the processor can natively execute. Or if the program wants to determine such a number at runtime, then there is no (entirely) portable way to do so.

OpenMP was created to fill that gap, by making the parallelization of selected segments of code easier.

3.2 Goal

In this first part of the assignment, you will use OpenMP to parallelize the matrix multiplication program from Project 1.

Based on the best i-j-k permutation from Project 1, write a program named mmm_omp.c that receives one command line argument, which defines the matrix order. Before quitting, the program displays the time it took to complete the matrix multiplication, as well as some checksum values to make sure that the result of the product is correct, and that the operands have not been changed.

4 Program #2: Heat distribution

4.1 Introduction

The static heat distribution of a room heated by a fireplace can be modeled using a stencil code. Stencil codes are a class of iterative algorithms which update array elements according to some fixed pattern, called stencil.

First, let us define our environment. Although a room is usually a 3-dimensional object, we will be modeling a 2-d square room only. The north wall will be the source of heat, while the three other walls will share the same lower temperature. The walls are never affected by the heat distribution themselves, they always emit a constant temperature.

If the entire computation of all the interior points is performed successively enough times, the difference between each iteration will eventually get closer to 0. At a certain threshold epsilon, we can consider that the heat distribution has become stable and the overall computation can stop.

4.2 Computation

The static heat distribution program follows three steps:

  1. Initialize the edges to their constant temperatures.

    Be careful that the corners should not be initialized twice. We assume that both corners of the north wall should get the fire temperature.

  2. Initialize the interior points to an initial value.

    Compute the mean value of all the edge points (again, do not count the corners twice) and use this mean value as initial value for all of the interior points.

  3. Compute the entire heat distribution of the room, by calculating the new temperature of all the interior points.

    Repeat the computation until the maximum difference between interior points of two successive generations falls below the specified threshold epsilon.

4.3 Serial code

Write program heat_distribution_serial.c that receives four command line arguments.

  • The first argument defines the resolution of the mesh of points (i.e., matrix order). This parameter can be between 1 and 2000.
  • The second argument specifies the temperature of the north wall (including both corners). It can be between 0 and 100.
  • The third argument specifies the temperature of the remaining walls. It can be between 0 and 100.
  • The fourth and last arguments gives the threshold epsilon that determines when the iterative computation should stop. It can be between 1e-6 and 100.

The program should time certain phases and display various information (by order): - Time the first two initialization phases (together) and print the value. - Compute and print the checksum of the heat map as well as the computed mean value. - For the checksum, the same file as for Project 1 is provided (matrix_checksum.c). - Time the entire computation phase and print when it’s done. - During the computation, and every power of two iteration, print the iteration number and the current maximum difference between iterations. - At the end of the computation, print the final iteration number and the final maximum difference. - Finally, the program shows the checksum of the final heat map, after the computation has completed.

4.3.1 Additional information and constraints

  • At each iteration, you have to compute a new heat map out of the previous heat map. There are (at least) two options to represent the new and old heat maps.

    1. Use two arrays, say h[][] and g[][]. h contains the old heat map, while g is used to compute the value of all the new interior points.

      g[i][j] = (h[i-1][j] + h[i+1][j] + h[i][j-1] + h[i][j+1]) / 4;

      After computing all the interior points and before the next iteration, g is entirely copied back into h.

    2. Use a 3-d array, such as h[2][][]. By dynamically switching between h[0] and h[1], you avoid having to copy one array into another.

      curr = 0;
next = 1;
while (...) {
    for (i = 1; i < N-1; i++)
        for (j = 1; j < N-1; j++)
            h[next][i][j] = (h[curr][i-1][j] + h[curr][i+1][j]
                    + h[curr][i][j-1] + h[curr][i][j+1]) / 4;
    curr = next;
    next = (curr + 1) % 2;
}
      Note that you do not necessarily need to have an actual 3-d array. You can also have a 2-entry 1-d array of pointers…
  • In order to compute the absolute difference between the successive values of the same interior point, you can use function fabs().
  • Your matrices should not be defined as global variables. They should exist only as local pointers in your main function, and passed to functions as arguments.

  • Although matrices are 2-D objects, they should be allocated as 1-D arrays to make sure that their memory layout is contiguous. Remember that it is easily possible to access a 1-D array as a 2-D structure (N being the number of columns):

    a[i][j] == a[i * N + j]

  • Make sure to separate your code into independent generic functions, called by the main function.
