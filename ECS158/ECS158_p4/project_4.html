3 Gaussian blur

A Gaussian blur is a 2-D convolution operator that is used to blur images in order to reduce detail and noise.

Technically, a Gaussian blur is represented as a matrix of weights, also called kernel or mask. The convolution is the process of adding each pixel of the image to its local neighbors, weighted by the matrix.

4 Program #1: serial implementation

4.1 Goal

Write program gaussian_blur_serial.c that receives two command line arguments.

  • The first argument specifies the name of the input PGM file.
  • The second argument specifies the name of the output PGM file.
  • The third argument specifies the sigma value to use for the blur. The sigma value should be considered valid only if the order of the resulting kernel matrix is not bigger than the input image’s width or height.

The program should time the computational phase and display the running time in the console. Finally, it should save the post-processed image.

$ ./gaussian_blur_serial city_256.pgm city_256_1.pgm 1
Running time: 0.021506 secs
$ ./gaussian_blur_serial ucd_pavilion_15mp.pgm ucd_pavilion_15mp_5.pgm 5
Running time: 18.379832 secs

Any errors should be reported on the standard error stream, and the program should stop with an error code of 1.

$ ./gaussian_blur_serial
Usage: ./gaussian_blur_serial <input_file> <output_file> <sigma>
$ ./gaussian_blur_serial doesntexist doesntexist_1 1
Error: cannot open file doesntexist
$ ./gaussian_blur_serial <(echo -e "not PGM") toto_1 1
Error: invalid PGM information
$ ./gaussian_blur_serial <(echo -e "P5\n2\n") toto_1 1
Error: invalid PGM information
$ ./gaussian_blur_serial <(echo -e "P5\n2 2\n255\n") toto_1 1
Error: invalid PGM pixels
$ ./gaussian_blur_serial city_256.pgm city_256_1.pgm 45
Error: sigma value too big for image size
$ ./gaussian_blur_serial city_256.pgm city_256_1.pgm toto
Error: invalid sigma value
$ echo $?
1

A working example of the program can be found at /home/cs158/public/p4/gaussian_blur_serial_reference.

Multiple PGM images of various sizes (from 256x256 to 15MP) can also be found in /home/cs158/public/p4/. Please, do not submit any of these images in your bundle.

4.2 Additional information and constraints

  • No need to use double-precision. Simple precision floating-point numbers should be enough and much faster (especially for the CUDA implementation).
  • For this project, we will consider the binary version of the PGM format.
    • The tag on the first line is P5 (instead of P2 for the ASCII version).
    • The next two lines are the same as in the ASCII version: dimensions of the image, and cutoff value (we can safely assume the cutoff will always be 255).
    • The rest of the file contain the pixel values, but in binary form. With a well-crafted fread() you can actually read the entire matrix of pixels directly in the image buffer. Same for generating the output, with fwrite().
  • When applying the kernel matrix on each pixel of the input image, make sure that you are staying without the bounds of the image. If pixels outside of the boundaries are needed, replace them by the nearest pixels of the corresponding edge.

5 Program #2: CUDA implementation

5.1 Goal

Write program gaussian_blur_cuda.cu that has the same behavior of the serial implementation but uses the GPU.

$ ./gaussian_blur_cuda city_256.pgm city_256_1.pgm 1
Running time: 0.082007 secs
$ ./gaussian_blur_cuda ucd_pavilion_15mp.pgm ucd_pavilion_15mp_5.pgm 5
Running time: 0.366971 secs

A working example of the program can be found at /home/cs158/public/p4/gaussian_blur_cuda_reference.

5.2 Additional information and constraints

  • Not all the CSIF computers have a Nvidia GPU card. Only computers numbered from pc21 to pc60 have an Nvidia GeForce GTX 960 card.
    • It is recommended not to use pc41, as it does not have the same GPU card as the other computer. It is equipped with an Nvidia GeForce GTX 970 which is a better card.
    • Make sure that no one else is using the same CSIF computer when running your CUDA-enabled program, as the GPU cannot be shared between programs.
    • For some reason, the first run on a given computer is always very slow. You have to run your program multiple times to start getting stable running times.
  • The convolution now has to be done by the GPU. To take advantage of the massively-parallel nature of the GPU, each CUDA thread should be in charge of computing only one pixel for the output image.
  • Apart from adding the CUDA function, you should only change the function corresponding to the computational phase in your serial implementation (i.e., the one that is timed).
    • Instead, it should allocate and prepare memory for the GPU, setup the computation grid, and then instantiate your CUDA function to run onto the GPU. Finally, the memory should be properly disposed and the result copied back into the main memory.

  • The compilation with nvcc is almost the same as with gcc except that you need to prefix all of the compiler flags with -Xcompiler:

    $ nvcc -Xcompiler -Wall -Xcompiler -Werror ... -o gaussian_blur_cuda gaussian_blur_cuda.cu -lm
